```ts
wpNuxt: {
  wordpressUrl: 'https://wordpress.wpnuxt.com',
  frontendUrl: 'https://demo.wpnuxt.com',
  composablesPrefix: 'useWP',
  defaultMenuName: 'main',
  enableCache: true,
  cacheMaxAge: 300,
  downloadSchema: true,
  logLevel: 3
},
```

## wordpressUrl

**Type:** `string` (required)

URL of the WordPress site

There is no default value for this option, so it's always required.

## frontendUrl

**Type:** `string` (optional)

URL of the Nuxt app

## composablesPrefix

**Type:** `string`
**Default:** `'useWP'`

Prefix for auto-generated composables.

## defaultMenuName

**Type:** `string`
**Default:** `'main'`

The default menu name to fetch when no menu name is specified in `useWPMenu()`.

## enableCache

**Type:** `boolean`
**Default:** `true`

Enable or disable caching for GraphQL queries. WPNuxt uses a two-layer caching strategy:
- In-memory LRU cache (L1) for hot queries
- HTTP cache layer (L2) with SWR (stale-while-revalidate)

See [Performance Optimization](/advanced/performance) for more details.

## cacheMaxAge

**Type:** `number`
**Default:** `300` (5 minutes)

Cache duration in seconds for the HTTP cache layer. This setting determines how long responses are cached before being revalidated.

Can also be set via environment variable:
```bash
WPNUXT_CACHE_MAX_AGE=600
```

## downloadSchema

**Type:** `boolean`
**Default:** `true`

Downloads the GraphQL schema and stores it on disk.

When setting this to false, the module will expect a `graphql.schema` file to be available.

You could first enable this, commit the schema file and then set `downloadSchema` to `false` to avoid having to do GraphQL introspection on each start of the application. This speeds up the build time of the application.

## logLevel

**Type:** `number`
**Default:** `3`

Controls logging verbosity. Higher values show more detailed logs.
