When building a headless WordPress site with WPNuxt, links in your WordPress content point to your WordPress domain by default. Clicking them would cause a full page reload instead of smooth client-side navigation. The <WPContent> component fixes this automatically.

How It Works

Wrap your rendered WordPress content in <WPContent> and any link pointing to your WordPress site gets intercepted. Instead of a full reload, WPNuxt uses Nuxt's navigateTo() for instant, SPA-style navigation.

No changes to your WordPress content are needed. Internal links are detected automatically based on your configured wordpressUrl.

Links to other websites are left untouched and open as you would expect:

  • WPNuxt on GitHub — source code and issues
  • Nuxt — the Vue framework powering it all
  • WordPress.org — where it all begins

What Gets Intercepted?

The <WPContent> component is smart about which clicks it handles.

Intercepted (client-side navigation):

  • Links matching your WordPress domain, like this internal link
  • Relative paths like this one

Not intercepted (default browser behavior):

  • External links like GitHub
  • Links with target="_blank" like this link that opens in a new tab
  • Ctrl+click or Cmd+click (user wants a new tab)
  • Email links like hello@example.com
  • Phone links like +1 234 567 890
  • Links with rel="external"

Button Blocks

Button blocks work too. Internal URLs get client-side navigation, external ones behave normally:

Visit Test Page (internal)
Visit WPNuxt Docs (external)

Usage

In your Nuxt page, wrap your content with the component:

<WPContent :node="post" />

When @wpnuxt/blocks is installed, <WPContent> automatically uses <BlockRenderer> for structured blocks. No extra setup needed.

Configuration

Link replacement is enabled by default. To disable it globally, add replaceLinks: false to your nuxt.config.ts:

wpNuxt: {
  wordpressUrl: 'https://wordpress.example.com',
  replaceLinks: false
}

You can also disable it per instance with the prop: <WPContent :node="post" :replace-links="false" />.
